# AGENTS — `@valve-tech/trueblocks-sdk`

Project-local notes for AI agents touching this package.

## Status

**Experimental.** Vertical-slice prototype demonstrating the
codegen-driven approach to wrapping chifra's HTTP API. Not yet
published. Licensed MIT (clean-room reimplementation against the
public OpenAPI spec — see Clean-room rule below).

## Architectural invariants

1. **Codegen-driven types.** `src/generated.ts` is auto-generated by
   `scripts/codegen.mjs` from the upstream OpenAPI 3.1 spec. Do NOT
   hand-edit it. Re-run `yarn codegen` if the upstream spec changes.
2. **Hand-curated public API.** `src/client.ts` and `src/verbs/*` are
   the consumer-facing layer. Keep them ergonomic, narrow, and
   independent of the generated types' raw shape.
3. **Browser/mobile safe.** `globalThis.fetch` only. No Node-only
   imports. The whole runtime is `fetch` + URL building.
4. **bigint at numeric boundaries.** Block numbers, timestamps, gas
   values, fee fields → bigint on the public API. Generated types
   carry whatever the spec said (often `string` for safety). The
   verb wrappers convert at the boundary.
5. **No silent fallbacks.** If a chifra endpoint returns an error or
   unexpected shape, throw a typed `TrueblocksError`.
6. **100/100/100/100 coverage gate.** Same as every other package in
   the toolkit. The vitest config enforces this with thresholds.
7. **Clean-room rule.** Do not read the GPL Go SDK source while
   writing this package. Shape comes from the OpenAPI spec only.

## Codegen pipeline

`scripts/codegen.mjs` pulls the OpenAPI 3.1 spec from the upstream
`TrueBlocks/trueblocks-core` repo at a pinned commit SHA, then runs
`openapi-typescript` to emit `src/generated.ts`. Keep the SHA pinned
— floating refs make codegen non-reproducible across builds.

To bump the spec version:

1. Verify the new commit's `openapi.yaml` shape via `gh api ...`
2. Update the SHA constant in `scripts/codegen.mjs`
3. Re-run `yarn codegen`
4. Diff `src/generated.ts` for breaking changes
5. Update verbs as needed
6. Bump CHANGELOG

## Test pattern

Each verb is a pure function taking a `request` callback. Tests pass
in a mock `request` so the verb logic is tested in isolation. The
client core (which builds the real `request` from `fetch`) has its
own tests with a mocked `fetch`.

This separation keeps verb tests fast (no URL building) and client
tests narrow (no per-verb logic).
